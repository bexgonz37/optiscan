## 1. Fixed this session (from live-testing feedback)

| Report | Root cause | Fix |
|---|---|---|
| "Short NVDA made no sense" | Stock callouts fired on a 10-second directional read with **no day-trend context** (options had `trendAlignedForTrade`; shares had nothing) | Day-trend alignment gate in `computeStockVerdict`: a SHORT while the stock is up >0.75% on the day (or LONG down >0.75%) becomes WAIT unless a level break provides structural proof. Tested. |
| "RIVN alerted after the move" | Discovery names sat through promotion (≤30s) + a 10-tick live warmup before they could trigger — structurally minutes late | New promotions now **seed their ring buffer from 1-min candle history** (`seedRingFromCandles`), so velocity/persistence windows are warm on arrival. | "Watch list flashes symbols" | List membership recomputed every tick; ordering was stabilized but membership wasn't | `lib/sticky-list.ts` membership dwell: symbols stay listed ~90s in a dimmed "cooling" state until they re-qualify or expire. Presentation-only. Tested. |
| "Conviction is confusing and jumpy" | Raw 0–100 recomputed every second from mixed sources, no explanation | Sampled every 15s, banded into words (LOW / BUILDING / STRONG / VERY STRONG) shown on the ring, full plain-English glossary entry behind an InfoTip (explicitly: *not* a win probability) |
| "Very few alerts" | Mostly by design (quality bar) but invisible | "Why didn't it alert?" panel on the live page surfaces the near-miss buffer: which symbol, which gate, values vs thresholds — proof the bar is enforced, not that the scanner is dead |

| Paper realism | Engine tracked prices but not full market state | Entry/exit snapshots now persist bid/ask/spread/IV/delta/gamma/theta/vega/OI/volume + entry reason; open positions expose live **unrealized P/L** ($/%) updated every sweep — the broker-adapter transition needs no schema change |

## 2. Architecture assessment

**Sound and worth keeping:** single-process design with globalThis state (correct for the product; documented); pure-function core (`zero-dte`, `paper-*`, `swing-score`, `health`, `near-miss`, `sticky-list`, `verdict-hold`) with I/O confined to engines/routes — this is why 325 tests run in ~7s with no mocks framework; central metered provider with hard caps; execution isolated behind the `Broker` interface; SQLite+WAL with guarded migrations.

**Watch items (accepted, documented):**
- `scanner-loop.ts` (~740 lines) is the one god-module. Acceptable while one person owns it; the extraction seam is already clean (pure helpers imported, near-miss/lock/budget externalized). Split `tick()` into pipeline stages if a second developer joins.
- Dual import styles: app code uses `@/lib/...`, cross-lib uses relative `./x.ts` (required for node-native tests). Convention documented here: **lib→lib imports are relative `.ts`; app→lib imports use the alias.**
- Duplicate concepts flagged in the original audit remain: `alert-format.js` vs `alert-format.ts`, `scan-core.ts` vs `scanner-loop.ts` (legacy `/scanner` page). Dead-weight, not dangerous. Candidate for deletion when the legacy page is retired.
- `tools/axiom-*` (390+ lines of scraped CSS/HTML dumps) are build artifacts committed to the repo — delete or move out of version control.

## 3. Quality gates status

Security: token header-only + timing-safe, no secrets in repo, quota hard-capped, single-instance lock — unchanged and healthy. Error handling: every engine sweep is try/caught with counters; API routes return typed failures; the health endpoint exposes liveness with 503 semantics. Logging: still console-based (structured JSON logging remains the top ops gap, carried on the roadmap). Testing: 45 test files; the pure cores are well covered; **UI components have no automated coverage** — that's the biggest testing gap and why §6 exists. Accessibility: InfoTips have aria labels/roles; a full a11y pass (focus order, contrast audit) has not been done. Mobile: bottom nav + responsive tooltips exist; the new /paper and /swing pages use the standard page scaffold but deserve a phone pass.

## 4. UI consistency

/paper and /swing now use the standard AppNav + panel + ledger + statgrid patterns with loading/empty/error states and glossary tooltips, and are registered in the nav rail and page-title map. Remaining polish gap vs the Axiom home screen: the home page has richer visual treatments (rings, tiles) than the utility pages — acceptable hierarchy (terminal vs tools), but if full parity is wanted, port `StatTile`/`Panel` onto /paper's statgrid (they're already used on `/`).
